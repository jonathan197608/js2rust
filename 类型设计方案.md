## 整体原则
取消现有类型推断机制，利用zig原生类型，不构造新类型，只保留JsAny这一个自定义的Union类型
最新的 JS 标准定义了 8 种数据类型：
七种基本数据类型：
Boolean。true 和 false。
null。一个表示空值的特殊关键字。（因为 JavaScript 是区分大小写的，所以 null 和 Null、NULL 或其他变体是不一样的。）
undefined。一个未定义值的顶级属性。
Number。整数或浮点数。例如，42 或 3.14159。
BigInt。任意精度的整数。例如，9007199254740992n。
String。表示文本值的字符序列。例如，"Howdy"。
Symbol。其实例是唯一且不可变的数据类型。
以及 Object
## 对应原则
undefined null BigInt 和 Symbol 禁止使用，报编译错误
对于Boolean、Number和String常量，JS与zig表达方式一致，无需转换
顶层赋值语句定义只允许常量表达式，变量报编译错误
尽量不做特殊处理，让zig出现编译错误，应通过修改JS的写法（保持语义相同）来避免编译错误。

## 具体设计需求
常量赋值语句无需指定类型，zig与JS写法一致。

对于各种操作符号，大致可以一一对应（不用转换操作数的类型），不能对应的，报编译错误
特殊处理一下加号，如果有一个操作数是字符串常量，则隐含要求另一个操作数也是字符串，如果都是常量，用 ++ 完成拼接，如果另一个是变量，
则运算结果转换为 std.ArrayList(u8)，调用变量的内置方法完成拼接并返回拼接后的字符串。如果两个都是变量，就默认是算术加法，不用特殊处理。

函数入参全部是 anytype，返回值类型用 @TypeOf内置函数根据所有return语句的表达式确定，例如：
fn add_or_sub(x: anytype, y: anytype) !@TypeOf(x+y, x-y) {
if (x>y) {
const tmp = x-y;
return tmp;
}
return x+y;
}
这里要注意，如果return表达式引入了中间变量，则使用中间变量的初始化表达式（比如上述的x-y，不能直接用tmp）
如果中间变量的初始化表达式（或者首个赋值语句）又包含了其他中间变量，则可以忽略，不用作为返回值类型推断的依据
如果入参没有使用，直接保持编译报错，不要额外处理
没有return语句的，返回值类型是!void
使用!类型，可以简化js throw语句的处理，直接返回对应错误即可，所有的函数调用，都加上try

函数入参假定不会有动态Object（但允许静态Object），动态Object只会是中间变量，动态Object不允许作为函数返回值使用（报编译错误）
动态Object使用StringHashMap(JsAny)，基于现有方案重新简化设计

常量数组直接对应zig的const数组
动态数组用 std.ArrayList(JsAny)实现，基于现有方案重新简化设计

函数体内中间变量处理逻辑如下：
变量赋值语句，需要指定变量类型，通过收集所有的变量赋值表达式，用以下方法确定：
变量必须赋值后使用，因此，变量赋值表达式如果出现未定义的变量，直接编译报错
接下来根据表达式的类型、操作符和操作数（函数入参没有固定类型，不参与）共同决定类型，可能的类型有：
值类型：i64 f64 bool []const u8 std.ArrayList(u8)
静态类型：数组 或 结构体
特殊约定，空数组表达式对应ArrayList(JsAny)，空对象表达式对应StringHashMap(JsAny)
出现其他一些zig不支持的表达式也无法对应转换的表达式，报编译错误

变量赋值语句统一这样写：
var c: []const u8 = undefined;
c = "abc";
var c: std.ArrayList(u8) = undefined;
c = std.ArrayList(u8).empty;

内存管理优化：
1、函数体内的中间变量，如果不用于返回值，函数返回时全部释放；
2、用于返回值的（只有字符串一种情况），采用全局分配器，等调用方主动释放。
