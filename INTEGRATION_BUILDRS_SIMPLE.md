# js2zig-build rs 集成方案（简化版）

> 目标：外部 Rust 项目通过 `build.rs` 一行调用，实现 JS → Zig → 静态库 → Rust FFI 的全自动编译集成。
> 简化要点：不保留 CLI、固定 `$OUT_DIR` 前缀、只支持多文件项目转译。

---

## 1. 整体架构

### 1.1 新增 2 个 crate

```
js2rust/                          # workspace root
├── js2zig-core/           [NEW]  # 核心转译库 (从 js2rustc 提取)
│   ├── Cargo.toml
│   └── src/
│       ├── lib.rs                # 公开 API
│       ├── pipeline.rs           # 编排逻辑 (js2rustc/main.rs → 此处)
│       ├── bridge.rs             # Bridge lib.rs 生成
│       └── runtime/              # include_str!() 嵌入的运行时
│
├── js2zig-build/          [NEW]  # build.rs helper (外部项目用)
│   ├── Cargo.toml
│   └── src/lib.rs                # compile_js() — build.rs 一键集成
│
├── js2rustc/              [删除] # CLI 工具 (不再保留)
├── js2rust-bridge/        [保持] # FFI 桥接 (保持现有结构)
├── js2rust-bridge-macro/  [简化] # proc-macro (固定 $OUT_DIR 前缀)
├── runtime/               [保持]  # Zig 运行时源文件
└── in/                    [保持]  # JS 测试文件
```

### 1.2 依赖关系

```
外部项目 Cargo.toml
  [build-dependencies]
  js2zig-build = "..."      ← build.rs 中调用

js2zig-build
  ↓ 依赖
js2zig-core
  ↓ 依赖 (compile-time: embed runtime/*.zig)
runtime/
  ↓ 生成 (写入 $OUT_DIR/js2zig/)
cabi_exports.json
  ↓ 读取 (compile time)
js2rust-bridge-macro
  ↓ 生成 (compile time)
FFI 绑定代码
```

| crate | 类型 | 说明 |
|-------|------|------|
| `js2zig-core` | lib | 核心转译库，嵌入 runtime |
| `js2zig-build` | lib (build-dep) | build.rs 一键集成 |
| `js2rust-bridge-macro` | proc-macro | FFI 绑定生成（固定 `$OUT_DIR`） |

## 2. js2zig-core API 设计

### 2.1 多文件项目转译（唯一接口）

```rust
// js2zig-core/src/lib.rs

/// 多文件项目配置
pub struct ProjectConfig {
    /// 项目名称（也是输出目录名和 Zig 库名）
    pub name: String,
    /// JS 源文件目录路径
    pub js_dir: PathBuf,
    /// 输出目录路径（通常是 $OUT_DIR）
    pub out_dir: PathBuf,
    /// Host 函数配置文件路径（可选）
    pub host_config: Option<PathBuf>,
}

/// 单个组的转译结果
pub struct GroupResult {
    /// 组名（如 "main", "utils"）
    pub name: String,
    /// 是否为测试组
    pub is_test: bool,
    /// C ABI 导出元数据 JSON
    pub cabi_exports_json: String,
    /// 诊断信息
    pub diagnostics: Vec<Diagnostic>,
    /// 生成的文件路径列表
    pub output_files: Vec<PathBuf>,
}

/// 项目转译结果
pub struct ProjectResult {
    /// 每个组的结果
    pub groups: Vec<GroupResult>,
    /// 全局诊断信息
    pub diagnostics: Vec<Diagnostic>,
}

/// 多文件项目转译：JS 目录 → Zig 项目 + cabi_exports.json
///
/// 等价于当前 main.rs 的完整流水线，但不调用 zig build，
/// 也不生成 bridge lib.rs（那是 js2zig-build 的职责）。
pub fn transpile_project(config: &ProjectConfig) -> Result<ProjectResult, Error>;
```

### 2.2 运行时嵌入

```rust
/// 运行时文件嵌入模块
pub mod runtime {
    /// 将嵌入的 runtime/*.zig 文件写入目标目录
    pub fn write_runtime(target_dir: &Path) -> Result<(), io::Error>;

    /// 获取所有 runtime 文件的 (filename, content) 列表
    pub fn embedded_files() -> &'static [(&'static str, &'static str)];
}
```

**嵌入策略**：使用 `include_str!()` 在编译时将 `runtime/*.zig` 嵌入 `js2zig-core` 二进制中。

```rust
// js2zig-core/build.rs (自动生成)
//
// 扫描 ../runtime/*.zig，生成 src/runtime/embed.rs

use std::fs;
use std::path::Path;

fn main() {
    let runtime_dir = Path::new("../runtime");
    let out_file = Path::new("src/runtime/embed.rs");

    let mut content = String::from("// auto-generated by build.rs — DO NOT EDIT\n\n");
    content.push_str("pub static RUNTIME_FILES: &[(&str, &str)] = &[\n");

    for entry in fs::read_dir(runtime_dir).unwrap() {
        let entry = entry.unwrap();
        let path = entry.path();
        if path.extension().is_some_and(|e| e == "zig") {
            let filename = path.file_name().unwrap().to_str().unwrap();
            content.push_str(&format!(
                "    (\"{}\", include_str!(\"../../runtime/{}\")),\n",
                filename, filename
            ));
        }
    }

    content.push_str("];\n");
    fs::write(out_file, content).unwrap();

    // rerun-if-changed
    println!("cargo:rerun-if-changed=../runtime/");
}
```

## 3. js2zig-build API 设计（外部项目用）

### 3.1 公开 API

```rust
// js2zig-build/src/lib.rs

/// build.rs 集成配置（简化：只保留必要字段）
pub struct JsBuildConfig {
    /// JS 源文件目录（相对于 CARGO_MANIFEST_DIR）
    pub js_dir: PathBuf,
    /// Host 函数配置文件（可选，相对于 CARGO_MANIFEST_DIR）
    pub host_config: Option<PathBuf>,
}

impl Default for JsBuildConfig {
    fn default() -> Self {
        Self {
            js_dir: PathBuf::from("js"),
            host_config: None,
        }
    }
}

/// build.rs 一键集成入口
///
/// 执行完整的 JS → Zig → 静态库 流水线，并输出 cargo link 指令。
/// 输出目录固定为 `$OUT_DIR/js2zig/`，Bridge Macro 自动使用此路径。
///
/// # Panics
/// 构建失败时 panic（build.rs 约定）
///
/// # 使用示例
/// ```no_run
/// // build.rs
/// fn main() {
///     js2zig_build::compile_js(Default::default());
/// }
/// ```
pub fn compile_js(config: JsBuildConfig) {
    match compile_js_inner(config) {
        Ok(()) => {},
        Err(e) => panic!("js2zig-build failed: {}", e),
    }
}

/// 内部实现（返回 Result）
fn compile_js_inner(config: JsBuildConfig) -> Result<(), Error> {
    let manifest_dir = PathBuf::from(env::var("CARGO_MANIFEST_DIR").unwrap());
    let out_dir = PathBuf::from(env::var("OUT_DIR").unwrap());
    let js_dir = manifest_dir.join(&config.js_dir);
    let zig_out = out_dir.join("js2zig");  // 固定前缀

    // 1. 转译 JS → Zig
    let project_config = js2zig_core::ProjectConfig {
        name: "js2zig_lib".into(),
        js_dir,
        out_dir: zig_out.clone(),
        host_config: config.host_config.map(|p| manifest_dir.join(p)),
    };

    let result = js2zig_core::transpile_project(&project_config)?;

    // 2. 打印诊断信息
    for group in &result.groups {
        for diag in &group.diagnostics {
            println!("cargo:warning=js2zig [{}] {}: {}", group.name, diag.kind, diag.message);
        }
    }

    // 3. 对每个非测试组执行 zig build
    let zig = find_zig()?;

    for group in &result.groups {
        if group.is_test { continue; }

        let group_dir = zig_out.join(&group.name);

        // zig build
        let status = std::process::Command::new(&zig)
            .arg("build")
            .current_dir(&group_dir)
            .status()
            .map_err(|e| format!("failed to run zig: {} (is Zig 0.16.0+ installed?)", e))?;

        if !status.success() {
            return Err(format!("zig build failed for group '{}'", group.name));
        }

        // 4. 输出 cargo link 指令
        let lib_dir = group_dir.join("zig-out").join("lib");
        println!("cargo:rustc-link-search=native={}", lib_dir.display());
        println!("cargo:rustc-link-lib=static={}", group.name);
    }

    // 5. Windows: 链接 MSVCRT 和 ntdll（Zig 生成的 COFF 需要）
    if cfg!(target_os = "windows") {
        println!("cargo:rustc-link-lib=dylib=ntdll");
    }

    // 6. 输出 rerun-if-changed 指令
    emit_rerun_directives(&config.js_dir, &manifest_dir);

    // 7. 设置环境变量供 bridge macro 使用（固定 $OUT_DIR/js2zig/ 前缀）
    println!("cargo:rustc-env=JS2ZIG_OUT_DIR={}", zig_out.display());

    Ok(())
}

/// 查找 Zig 编译器
fn find_zig() -> Result<PathBuf, String> {
    // 1. 检查 PATH
    if let Ok(output) = std::process::Command::new("zig").arg("version").output() {
        if output.status.success() {
            return Ok(PathBuf::from("zig"));
        }
    }
    // 2. 常见安装位置
    let candidates = [
        "C:\\zig\\zig.exe",
        "/usr/local/bin/zig",
        "/opt/homebrew/bin/zig",
    ];
    for path in candidates {
        if Path::new(path).exists() {
            return Ok(PathBuf::from(path));
        }
    }
    Err(
        "Zig compiler not found. Install Zig 0.16.0+ and ensure it's in PATH.\n\
         Download: https://ziglang.org/download/".into(),
    )
}

/// 输出 rerun-if-changed 指令
fn emit_rerun_directives(js_dir: &Path, manifest_dir: &Path) {
    let js_dir_abs = manifest_dir.join(js_dir);

    // 监控整个 JS 目录
    println!("cargo:rerun-if-changed={}", js_dir_abs.display());

    // 逐文件监控（确保新增/删除文件也触发重建）
    if let Ok(entries) = std::fs::read_dir(&js_dir_abs) {
        for entry in entries.flatten() {
            let path = entry.path();
            if path.extension().is_some_and(|e| e == "js") {
                println!("cargo:rerun-if-changed={}", path.display());
            }
        }
    }
}
```

## 4. Bridge Macro 简化（固定 $OUT_DIR 前缀）

### 4.1 问题

当前 `js2rust_bridge_macro` 用硬编码路径 `"out/main/cabi_exports.json"` 查找元数据。对于外部项目：
- `out/` 不存在
- `cabi_exports.json` 在 `$OUT_DIR/js2zig/{group}/` 中

### 4.2 简化方案：固定 $OUT_DIR/js2zig/ 前缀

Bridge Macro 内部**固定**使用 `$OUT_DIR/js2zig/` 作为前缀，用户只需指定组名。

```rust
// js2rust-bridge-macro/src/lib.rs

/// 简化版：只需指定组名，前缀固定为 $OUT_DIR/js2zig/
///
/// # 使用示例
/// ```ignore
/// // 自动查找 $OUT_DIR/js2zig/main/cabi_exports.json
/// js2rust_bridge!("main");
/// ```
pub fn js2rust_bridge(input: proc_macro::TokenStream) -> proc_macro::TokenStream {
    let group_name = parse_macro_input!(input as LitStr).value();

    // 固定前缀: $OUT_DIR/js2zig/{group_name}/cabi_exports.json
    let out_dir = env!("OUT_DIR");
    let json_path = PathBuf::from(out_dir)
        .join("js2zig")
        .join(&group_name)
        .join("cabi_exports.json");

    if !json_path.exists() {
        panic!(
            "js2rust_bridge: cabi_exports.json not found at {}\n\
             Hint: Did you run js2zig-build in build.rs?",
            json_path.display()
        );
    }

    let json_content = fs::read_to_string(&json_path)
        .unwrap_or_else(|e| panic!("js2rust_bridge: failed to read {}: {}", json_path.display(), e));

    let exports: Vec<CabiExport> = serde_json::from_str(&json_content)
        .unwrap_or_else(|e| panic!("js2rust_bridge: invalid JSON in {}: {}", json_path.display(), e));

    generate_ffi_bindings(&exports, &group_name)
}
```

**注意**：`env!("OUT_DIR")` 在 proc-macro 中展开为**编译 macro 的 crate** 的 `$OUT_DIR`，而非使用 macro 的 crate。

**正确方案**：使用 `std::env::var("OUT_DIR")` 运行时获取，但 proc-macro 在**编译期**运行，此时 `OUT_DIR` 环境变量指向**当前 crate 的 OUT_DIR**。

**最终方案**：通过 `cargo:rustc-env` 在 build.rs 中设置 `JS2ZIG_OUT_DIR`，macro 读取此环境变量。

```rust
/// 最终版：读取 JS2ZIG_OUT_DIR 环境变量（由 js2zig-build 设置）
pub fn js2rust_bridge(input: proc_macro::TokenStream) -> proc_macro::TokenStream {
    let group_name = parse_macro_input!(input as LitStr).value();

    // 读取 js2zig-build 设置的 JS2ZIG_OUT_DIR 环境变量
    let js2zig_out_dir = std::env::var("JS2ZIG_OUT_DIR")
        .unwrap_or_else(|_| {
            panic!(
                "js2rust_bridge: JS2ZIG_OUT_DIR not set.\n\
                 Hint: Add js2zig-build to [build-dependencies] and call js2zig_build::compile_js() in build.rs"
            )
        });

    let json_path = PathBuf::from(&js2zig_out_dir)
        .join(&group_name)
        .join("cabi_exports.json");

    // ... 读取 JSON 并生成 FFI 绑定 ...
}
```

## 8. 实施步骤

### Step 1: 新建 js2zig-core crate (2-3h)

1. `cargo init --lib js2zig-core`
2. 添加到 workspace members
3. 将 `js2rustc/src/` 中的模块移动到 `js2zig-core/src/`
4. 定义 `ProjectConfig`, `ProjectResult`
5. 实现 `runtime/embed.rs`（`include_str!()` 嵌入所有 runtime 文件）

### Step 2: 删除 js2rustc CLI (0.5h)

1. 从 workspace 移除 `js2rustc`
2. 删除 `js2rustc/` 目录

### Step 3: 新建 js2zig-build crate (2h)

1. `cargo init --lib js2zig-build`
2. 实现 `compile_js()`
3. 实现 Zig 编译器查找、cargo link 指令输出、rerun-if-changed
4. 设置 `JS2ZIG_OUT_DIR` 环境变量

### Step 4: 简化 bridge macro (1h)

1. `resolve_json_path()` 固定使用 `JS2ZIG_OUT_DIR` 环境变量
2. 用户只需指定组名

### Step 5: 发布到 crates.io (1h)

1. 更新所有 `Cargo.toml` (`authors`, `license`, `description`, etc.)
2. `cargo publish` 按顺序发布 4 个 crate

### Step 6: 验证 (1h)

1. `cargo test` — 所有现有测试不变
2. `cargo clippy` — 零警告
3. 创建外部项目测试：`cargo add js2zig-build --build`，验证端到端流程
4. 提交

**总预估：7-9h（~2 天）**

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| `include_str!()` runtime 文件增/减 | 清单不同步 | `js2zig-core/build.rs` 自动扫描 `runtime/` 目录生成清单 |
| 跨平台 Zig 路径 | Windows/macOS/Linux 安装位置不同 | `find_zig()` 多路径探测 + 清晰的 panic 消息 |
| proc-macro 编译期 `OUT_DIR` | `env!("OUT_DIR")` 指向错误目录 | 使用 `std::env::var("JS2ZIG_OUT_DIR")` 运行时获取 |
| Zig 0.16.0 API 变更 | 未来 Zig 版本可能不兼容 | `build.zig.zon` 锁定 `minimum_zig_version = "0.16.0"` |
| crates.io 发布 | `include_str!()` 需要 runtime 文件在 crate 包内 | `Cargo.toml` 的 `include` 字段显式包含 `runtime/` |
| 编译时间 | build.rs 中全量重建较慢 | Cargo `rerun-if-changed` 确保只在 JS 变更时重建 |
