feat(zigir): descriptive CompileErrors for unsupported statements/expressions

- Known-unsupported constructors (ArrayBuffer, SharedArrayBuffer, Function,
  Promise, WeakMap, WeakSet, DataView) now produce NewConstructor::Unsupported
  which the Emitter renders as a proper @compileError
- Nested ClassDeclaration produces descriptive CompileError with class name
- ClassExpression produces descriptive CompileError with class name
- DebuggerStatement produces IrStmt::Comment("debugger")
- ImportDeclaration / ExportDefaultDeclaration produce descriptive CompileErrors
- Generic unknown NewExpression catch-all now includes the constructor name
