### 3.2 核心能力

✅ **已支持**:
- ES5 完整语法（除 `with` 语句）
- ES6+ 核心特性（箭头函数、闭包捕获、类、模板字符串、解构、rest/spread、可选链 `?.`）
- 异步编程（async/await + Io 模式）
- 错误处理（throw → error.JsThrow + try-catch）
- 类型推断（3 层规则 + 约束求解）
- 闭包捕获（自动生成 Closure 结构体，value/reference capture）
- C ABI 桥接（Rust ↔ Zig，双 Arena 全局分配器自动管理内存）
- `for-in` 静态/动态对象

## 4. 已知限制

### 4.1 语言特性限制

| 限制 | 说明 | 优先级 | 状态 |
|------|------|--------|------|
| 解构默认值 | 跳过默认值，始终使用 `undefined` | P2 | 📋 待解决 |
| 多 spread 合并 `{ ...a, ...b }` | 不支持 | P2 | 📋 待解决 |
| 嵌套函数声明 | 报错，需重构为顶层 | P2 | 📋 待解决 |
| Promise API | 不支持，使用 `async/await` 替代 | P3 | 📋 待解决 |
| Generator / `yield` | 不支持 | P3 | 📋 待解决 |
| 动态 `import()` | 不支持，使用静态 `import` | P3 | 📋 待解决 |
| 私有字段 `#field` | 不支持 | P3 | 📋 待解决 |
| 类表达式 | 不支持 | P3 | 📋 待解决 |
| 标签模板 | 不支持 | P3 | 📋 待解决 |
| JSX | 不支持，使用 `createElement()` 调用 | P3 | 📋 待解决 |
| `with` 语句 | JS 严格模式已废弃 | - | ❌ 不计划支持 |
| `debugger` 语句 | 不支持 | - | ❌ 不计划支持 |

### 4.2 类型系统限制

| 限制 | 说明 | 优先级 | 状态 |
|------|------|--------|------|
| Class 字段类型 | 所有字段类型硬编码为 `i64` | P2 | 📋 待解决 |
| TypeScript 泛型 | 不支持 | P3 | 📋 待解决 |
| `interface` / `type` alias | 不支持 | P3 | 📋 待解决 |
| 复杂联合类型 | 导出函数参数不支持联合类型（Rule 8 报错） | P3 | 📋 待解决 |

### 4.3 运行时限制

| 限制 | 说明 | 优先级 | 状态 |
|------|------|--------|------|
| 正则表达式引擎 | 需引入 C 库（如 `pcre2`） | P2 | 📋 待解决 |
| `Math.random()` 安全性 | 使用 `std.crypto.random`，但为真随机 | P3 | 📋 待解决 |
| 大文件 JS | 转译器性能未优化 | P3 | 📋 待解决 |
| 错误信息 | 可改进为附加源位置 + 建议 | P3 | 📋 待解决 |

## 6. 贡献指南

### 6.1 任务认领流程

1. 查看本文档的 P1/P2/P3 任务列表
2. 选择任务并在状态列标记为 `🚧 进行中`
3. 在任务说明中添加认领日期和预计完成日期
4. 完成后更新状态为 `✅ 已完成` 并添加完成日期

### 6.2 提交要求

- 每个任务完成后需通过所有现有测试（145 个测试）
- 新功能需添加测试用例（Rust 测试或示例项目）
- 提交信息需包含任务编号和说明（如 `feat: implement TypedArray.set() (P1)`）
